DETAILS

  • This project returns a JSON converting a unit from metric to imperial vice versa
  • For a valid measurement input, the api returns a JSON having the structure:

    • {"initNum": <number input> ,"initUnit": <unit inputted>, "returnNum": <number converted> , "returnUnit": <unit converted>, "string": <information>}

  • For an invalid dates string, the api returns a JSON having the structure:

    • {"unix": null, "utc" : "Invalid Date" }

  • If no date is provided it returns a JSON having the current date.

    • EXAMPLES (click on the links)

  • Date Input
    • [root_url]/api/timestamp/2018-11-20

    JSON output: {"unix":1542672000000,"utc":"Tue, 20 Nov 2018 00:00:00 GMT"}

  • UNIX Input
    • [root_url]/api/timestamp/1545264000000

    JSON output: {"unix":1545264000000,"utc":"Thu, 20 Dec 2018 00:00:00 GMT"}

  • Invalid Date Input
    • [root_url]/api/timestamp/alskfy(87

    JSON output: {"unix":null,"utc":"Invalid Date"}

  • No date Input
    • [root_url]/api/timestamp/

    JSON output: current date generated by new Date();
